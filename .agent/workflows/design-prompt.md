# 디자인 프롬프트 생성

구현 AI가 자율적으로 UI를 설계·구현할 수 있도록, **사용자가 보고 조작하는 것**만 선언적으로 기술하는 문서를 생성한다.

## DESIGN.md 선행 참조

- 대상 프로젝트 root `DESIGN.md`가 있으면 prompt 작성 전에 먼저 읽는다.
- `DESIGN.md`가 없으면 `design-system` skill로 생성을 제안하거나, 부재를 사용자에게 보고한다.
- `DESIGN.md`의 색상/폰트/spacing 토큰을 prompt 본문에 복사하지 않는다. 스타일은 `DESIGN.md`가 담당하고 prompt는 보이는 정보/가능한 조작을 담당한다.

## 작성 원칙

### 선언적 기술
- **적는다**: "plan 목록이 보인다 (제목, 상태, 진행률)"
- **안 적는다**: "좌측에 카드형 패널로 배치한다"

### 금지 항목
- 위치 지정 (상단/좌측/우측/하단 등)
- UI 패턴 지정 (탭/필터/모달/드로어/카드/패널/그리드 등)
- 구현 기술 언급 (프레임워크, 라이브러리, API 스키마, 파일 경로 등)
- "기존", "현재", "사용자가~" 같은 현상 설명
- DESIGN.md의 색상/폰트/spacing 등 스타일 토큰을 prompt 본문에 복사

### 서술 형태
- "X가 보인다", "X를 할 수 있다", "X 조건에서 노출된다" — 이 세 형태만 사용

## 실행 순서

1. 대상 프로젝트 root `DESIGN.md` 존재 확인 → 있으면 읽기, 없으면 부재 보고
2. 사용자에게 대상 페이지와 원하는 변경 확인
3. 보이는 정보와 가능한 조작을 선언형 문장으로 작성
4. 체크리스트 검토 후 저장

## 체크리스트

- [ ] DESIGN.md가 있으면 먼저 참조했는가?
- [ ] DESIGN.md의 스타일 토큰을 본문에 복사하지 않았는가?
- [ ] 모든 항목이 "X가 보인다" 또는 "X를 할 수 있다" 형태인가?
- [ ] 위치를 지정하고 있지 않은가?
- [ ] UI 패턴을 강제하고 있지 않은가?
- [ ] 구현 기술을 언급하고 있지 않은가?
- [ ] "기존", "현재" 표현이 없는가?

## 저장
